  • PR

    • Landing Page

    • FAQs

      Last week's feedback had a bunch of questions that we wanted to answer. Check out the FAQs here : https://meterele.app#section-5. We shall be using this section to answer more questions that come up during the next month.

    • Outreach - NEC

      We contacted NEC, a Japanese company that operates the bus fleet platform, in a bunch of cities like Ahmedabad, Hubli, etc. and had over 4 hours of discussion with them. They are extremely excited to test run Saturn platform and we have our first collaboration deal! :D . We are working on a mutually agreeable set of features that we want to test on actual users. We're meeting the officials from NEC on the last weekend of April, to demo the platform. We're also finalizing a data sharing agreement.



  • Product

    • Customer App

      We've built a react native app that does the following : Enter destination (Google maps places integrations done), displaying journey options, selecting the preferred journey, entering the meter mele amount, entire auto-rickshaw booking flow, journey completion.


    • Driver App

      Even here, we've built a react native app. The features built are primiarily for autorickshaw drivers. On the app, they can view all the bids (total meter cost + metermele extra cost) and accept the bid that suits their estimation of a "fair fare". Once they accept a bid, it follows the same ride-hailing flow we're all used to.

    • Server

      Both these apps have been wired to the server and with dummy data, can go through the entire flow, like how a regular user would.
